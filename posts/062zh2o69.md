一款由 GitHub API 驱动的优雅现代个人博客系统。ZenBlog 采用"存储计算分离"架构，您的代码存放在 `main` 分支，而内容则存储在干净独立的 `data` 分支。

## 核心特性

- **独立存储**：数据提交到专用的 `data` 分支，保持 `main` 分支整洁
- **原子提交**：内容和索引更新合并为单个 Git 提交确保完整性
- **Markdown 优先**：支持 GFM 的高质量文章渲染
- **现代化界面**：基于 React、Tailwind CSS、Framer Motion 和 Lucide 图标构建
- **零后端**：完全无服务器，完全由 GitHub Git API 驱动
- **可编辑关于页面**：完全可自定义的关于页面，包含兴趣、技能、作品、音乐、联系方式和游戏
- **站点设置**：可配置站点名称、图标和描述
- **国际化**：支持中文和英文
- **主题支持**：亮色、暗色和自动主题模式

## 快速开始

1. **安装依赖**：
   ```bash
   npm install
   ```

2. **本地运行**：
   ```bash
   npm run dev
   ```

3. **配置**：
   - 在 `public/` 目录创建 `config.json` 文件（参见下方配置章节）
   - 前往应用内的 **设置** 页面，提供您的 GitHub 个人访问令牌（PAT）

4. **部署**：
   只需将代码推送到 GitHub，并在 `main` 分支启用 GitHub Pages（使用提供的 GitHub Action）。

## 配置

ZenBlog 使用存储在 `main` 分支的配置文件（`config.json`）来确定从哪个仓库获取数据。这使得访问者能够自动发现您的博客配置。

### 初始设置

1. **创建 config.json**：在 `public/` 目录创建 `config.json` 文件，结构如下：
   ```json
   {
     "owner": "您的 GitHub 用户名",
     "repo": "您的仓库名称",
     "branch": "data"
   }
   ```

2. **部署 config.json**：构建和部署项目时，`config.json` 文件将作为静态文件包含在内。访问者可以通过 `/config.json` 自动访问您的博客数据。

3. **配置令牌**：前往应用内的 **设置** 页面，为写入操作提供您的 GitHub 个人访问令牌（PAT）。令牌仅存储在 localStorage 中，永远不会提交到仓库。

**注意**：
- `config.json` 文件仅包含公开信息（所有者、仓库、分支）
- 设置页面中的仓库所有者、仓库名称和数据分支字段为只读，显示来自 `config.json` 的值
- GitHub 令牌仅存储在 localStorage 中，永远不会提交到仓库

### 设置概览

设置页面包含三个主要部分：

1. **GitHub 存储配置**：
   - 个人访问令牌（可编辑，本地存储）
   - 仓库所有者（只读，来自 config.json）
   - 仓库名称（只读，来自 config.json）
   - 数据分支（只读，来自 config.json）

2. **作者资料**：
   - 显示名称
   - 个人简介
   - 头像 URL
   - GitHub 链接

3. **站点设置**：
   - 站点名称（用于页面标题和导航栏）
   - 站点图标（用作 favicon）
   - 站点描述（用于 SEO 元描述）

所有资料和站点设置都保存在数据分支的 `data/profile.json` 中。
